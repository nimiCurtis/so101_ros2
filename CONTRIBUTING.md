# Contributing to so101_ros2

We welcome contributions from the community to enhance the functionality,
fix bugs, and improve documentation. By contributing to this repository, you agree
to abide by the following guidelines:

## How to Contribute

1. Fork the Repository: Start by forking the repository to your own GitHub account.

2. Create a Branch: Create a new branch for your feature or bug fix. Use a descriptive name for the branch.

3. Make Changes: Implement your changes in the new branch. Ensure that your code adheres to the project's coding standards and conventions.

4. Test Your Changes: Thoroughly test your changes to ensure they work as intended and do not introduce new issues.

5. Update Documentation: If your changes affect the documentation, please update the relevant documentation files accordingly.

6. Precommit Your Changes: Before committing, run the pre-commit hooks to ensure code quality and consistency. You can do this by executing:

   ```bash
   pre-commit run --all-files
   ```

7. Commit Your Changes: Commit your changes with clear and concise commit messages that describe the purpose of the changes.

8. Push to Your Fork: Push your changes to your forked repository on GitHub.

9.  Create a Pull Request: Navigate to the original repository and create a pull request from your forked repository. Provide a detailed description of the changes you made and the reasons for them.
